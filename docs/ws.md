# WebSocket Contract

This websocket contract is optional. If `VITE_WS_URL` is not set, the frontend does not open realtime connections and uses the HTTP API for ordinary load/save behavior.

When `VITE_WS_URL` is set, the frontend expects a Hocuspocus/Yjs-compatible websocket server. The frontend does not include or run a websocket server.

## General Rules

| Rule | Contract |
| --- | --- |
| Enable flag | `VITE_WS_URL` must be set in the frontend environment. |
| Protocol | Hocuspocus/Yjs websocket protocol. |
| Transport | WebSocket. |
| Payload format | Yjs protocol messages, not JSON application messages. |
| Authentication | None required by the frontend by default. Add auth at the websocket server or provider configuration later if needed. |
| Fallback behavior | If `VITE_WS_URL` is unset, the editor uses REST load/save. |
| Server ownership | External API/realtime service owns the websocket server. This frontend repo does not. |

## Endpoints

### WebSocket `VITE_WS_URL`

Connects the frontend to an optional Hocuspocus/Yjs-compatible realtime service.

#### Parameters

| Name | Location | Type | Required | Description |
| --- | --- | --- | --- | --- |
| `VITE_WS_URL` | Environment | string | Yes, for realtime | Full websocket URL. Example meaning: external realtime server base URL. |
| Document name | Hocuspocus provider option | string | Yes | The Yjs document name. See document-specific sections below. |
| Document payload | WebSocket protocol | Yjs update | Yes | Binary Yjs/Hocuspocus protocol payload. |

#### Response

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| Connection status | Hocuspocus provider event | Yes | Provider must emit normal connection/sync lifecycle events. |
| Synced state | Hocuspocus `synced` event | Yes | Frontend waits for synced state before enabling realtime editing. |
| Awareness states | Yjs awareness map | No | Used for presence display when available. |

### Document `{nodeId}`

Realtime document for markdown content editing. The document name is the file node id.

#### Parameters

| Name | Location | Type | Required | Description |
| --- | --- | --- | --- | --- |
| `nodeId` | Document name | string | Yes | File node id. |
| `codemirror` | Yjs text field | Y.Text | Yes | Shared markdown document content. |
| Awareness user | Awareness state | object | No | Local user metadata for collaborative cursors/presence. |
| Awareness user `id` | Awareness state | string | Yes, when awareness user is sent | Stable local client id from browser local storage. |
| Awareness user `name` | Awareness state | string | Yes, when awareness user is sent | Display name. Current frontend sends `Local editor`. |
| Awareness user `color` | Awareness state | string | Yes, when awareness user is sent | Hex display color. Current content editor sends `#000000`. |

#### Response

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `codemirror` updates | Y.Text updates | Yes | Remote markdown content changes applied to the editor. |
| Awareness updates | Yjs awareness updates | No | Remote user metadata used by Yjs collaboration extensions. |
| Undo origin | Awareness client id | No | Used by the frontend undo manager when available. |

### Document `project-presence-{projectId}`

Realtime document for file tree refresh signals and active-user presence.

#### Parameters

| Name | Location | Type | Required | Description |
| --- | --- | --- | --- | --- |
| `projectId` | Document name segment | string | Yes | File scope identifier. |
| `project-presence-{projectId}` | Document name | string | Yes | Presence document name. |
| `sync-state` | Yjs map field | Y.Map number | Yes | Shared map used to signal remote tree changes. |
| `sync-state.lastUpdate` | Yjs map key | number | No | Timestamp-like value set when the local file tree changes. |
| Awareness user | Awareness state | object | No | Local user metadata for active-user display. |
| Awareness user `id` | Awareness state | string | Yes, when awareness user is sent | Stable local client id from browser local storage. |
| Awareness user `name` | Awareness state | string | Yes, when awareness user is sent | Display name. Current frontend sends `Local editor`. |
| Awareness user `color` | Awareness state | string | Yes, when awareness user is sent | Hex display color generated by the frontend. |

#### Response

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `sync-state` remote change | Y.Map update | No | When a remote update is observed, the frontend invalidates the file tree query. |
| Awareness states | Map of awareness states | No | Frontend displays active users from states with `user.id`. |
| Connection cleanup | Provider lifecycle | Yes | Server must tolerate clients disconnecting and reconnecting as routes/files change. |

### Shared Chunk State `chunk-state`

Optional Yjs map used inside chunk editing mode. This state is stored in the same file document as `{nodeId}`.

#### Parameters

| Name | Location | Type | Required | Description |
| --- | --- | --- | --- | --- |
| `chunk-state` | Yjs map field | Y.Map | No | Map for chunk editor state. |
| `chunk-state.splits` | Yjs map key | array of number pairs | No | Chunk split ranges. Each pair is start and end character offset. |

#### Response

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `chunk-state.splits` remote change | array of number pairs | No | Frontend applies remote chunk split changes to the chunk editor. |

## Connection Lifecycle

| Step | Document | Frontend Behavior | Server Requirement |
| --- | --- | --- | --- |
| Realtime disabled | None | Does not connect to websocket. Uses REST save. | No websocket server required. |
| Open file | `{nodeId}` | Creates a Hocuspocus provider for the file document. | Accept document named by node id. |
| File synced | `{nodeId}` | Enables realtime editor and writes awareness user state. | Emit synced lifecycle event after document sync. |
| Switch file | Previous `{nodeId}` then new `{nodeId}` | Destroys old provider and opens the new file document. | Clean up disconnected clients. |
| Open file scope | `project-presence-{projectId}` | Creates a presence provider for active users and tree refresh signals. | Accept presence document names. |
| Tree changed | `project-presence-{projectId}` | Sets `sync-state.lastUpdate`. | Broadcast Yjs map update to other clients. |
| Remote tree change | `project-presence-{projectId}` | Invalidates file tree query. | Preserve transaction locality so remote changes are detectable. |
| Unmount | Any active provider | Destroys provider. | Clean up awareness state and connection resources. |

## Required Server Capabilities

| Capability | Required | Description |
| --- | --- | --- |
| Hocuspocus/Yjs protocol support | Yes | The current frontend uses `@hocuspocus/provider`. |
| Multiple named documents | Yes | File documents and presence documents use different names. |
| Y.Text persistence for `codemirror` | Yes, if realtime content should persist | Required for collaborative markdown content. |
| Y.Map support for `sync-state` | Yes, for presence document | Required for tree refresh signaling. |
| Awareness support | Yes, for presence | Required for active-user display and collaboration metadata. |
| Cross-origin websocket access | Yes | Server must accept connections from the frontend origin. |
| REST fallback compatibility | Yes | REST API remains the default path when websocket is disabled. |

## Non-Goals

| Item | Contract |
| --- | --- |
| In-repo websocket server | Not included. |
| Required realtime collaboration | Not required. |
| Required websocket authentication | Not required by the current frontend. |
| JSON websocket messages | Not used by the current frontend. |
